# 설명

리액트를 이용해 게시판을 만들어 봤다. 

로그인을 하지 않아도 게시판 글을 읽을 수 있고 작성을 하려면 로그인을 해야한다. 

로그인 화면엔 아이디 및 비밀번호를 찾을 수 있고 회원가입이 가능하다. 

회원가입 또는 게시글 작성 시 몽고db에 저장이 된다.

회원 정보는 닉네임만 수정이 가능하다.

게시글은 수정 및 삭제가 가능하다. 

왼 쪽 카테고리에는 각 게시글의 종류에 따라 분류해 볼 수 있다.


# 사용 기술
react, node.js, 몽고db, express, axios, styled-component


# 이번 프로젝트에서 중요하게 생각한 기능

처음으로 서버와 연결해 데이터를 저장했다. 

node.js의 express를 통해 서버를 만들었고 axios를 이용해 데이터를 주고 받았다.

받은 데이터는 몽고db에 저장했다.

여기서 몽고db를 사용한 이유는 데이터베이스를 실제로 적용한 프로젝트는 이번이 처음이었고 NOSQL 데이터베이스라 우리가 흔히 아는 
키-값으로 데이터를 저장할 수 있어 접근이 쉬웠다. 

처음 적용하다보니 첫 설정에서 시간도 많이 걸렸다. 

# 아쉬운 점

다 만들고 컴포넌트를 보니 리액트의 장점을 전혀 활용하지 못했다. 

하나의 컴포넌트로로 만들어 관리할 수 있는 똑같은 기능들도 컴포넌트 안에 하나하나 만들었던 것이다.

그러다 보니 폴더 구조도 복잡해 졌다. 

이 부분과 연관된 것이 아키텍처의 필요성이다. 

나는 전체적인 틀을 잡고 어떠한 기능들만 넣어야지 까지만 생각을 했다.

하지만 여기서 각 기능들에 필요한 기술들을 자세히 작성하고 자주 사용되는 기능은 하나의 컴포넌트를 만들 것까지 생각을 했어야 했다.

다음 프로젝트는 제대로 아키텍처를 설계하고 진행할 예정이다.

그리고 디자인 적 부분이 많이 부족하다. 

물론 기능이 더 중요하지만 디자인이 많이 아쉽다. 

# 개선할 점

프론트엔드는 만들고 끝이 아닌 사용자의 입장에서 계속 생각해 좀 더 나은 편의성을 제공해야 한다고 들었다. 

즉 코드의 리팩토링 및 디자인 개선을 할 생각이다.

1. 우선적으로 폰트를 전역으로 관리할 예정이다.

2. 자주 사용되는 기능은 하나의 컴포넌트로 만들 예정이다.

3. 디자인 감각이 늘게 되면 디자인 컨셉은 유지한 채 보강할 생각이다.

4. 데이터를 주고 받을 때 예외처리를 세분화할 생각이다.
    - axios와 node.js를 프로젝트를 진행하면서 공부한 것이라 예외처리를 엉성하게 했다.
